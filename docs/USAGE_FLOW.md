# 使用流程（v1.0.0）

## 1. 启动项目

### 1.1 本地链路（开发推荐）
1. 在项目根目录执行：`npm run dev:local`（或双击 `start-local.bat`）
2. 脚本会做：
   - Node/npm 检查
   - 依赖缺失时执行 `npm install`
   - 尝试释放 `5173/4000` 端口
3. 浏览器访问：`http://localhost:5173`

### 1.2 云端链路（体验站）
1. 在服务器项目根目录执行构建并启动后端服务
2. 浏览器通过域名访问前端
3. 前端请求同域 `/api/*`，由反代转发到后端 `127.0.0.1:4000`

## 2. 进入游戏前（Setting）
1. 点击左上角 `Setting`
2. 在“会话配置”确认：
   - 部署链路（本地/云端）
   - 世界观
   - 难度
   - 本地模式下填写本地 API Key 与模型参数
3. 点击“确认本局环境”
4. 状态显示“已确认”后关闭 Setting

## 3. 创建角色并开局
1. 输入人设提示词（长度 `4~500`）
2. 分配五维属性：`智力/魅力/家境/气运/体魄`
3. 属性规则：
   - 每项 `0~10`
   - 总和必须等于本局抽到的 `talentPointTotal`
4. 选择天赋卡（数量必须在 `selectedCardMin~selectedCardMax`）
5. 点击“开始游戏”

## 4. 流式推进顺序（当前默认）
开局与推进默认使用 NDJSON 流式事件：

1. `started`：先进入局内页面（此时不下发 milestone）
2. `timeline`：逐条推送年份文本（AI 生成完成后才发）
3. `meta`：附带本段推进元数据
4. `milestone`：若触发抉择，AI 文案就绪后下发
5. `done`：本轮推进结束

说明：
- `started` 阶段不下发抉择，避免前端先挂 fallback 选项文案。
- 前端“抉择历史”会在收到该抉择年的 `timeline` 文本后再挂载记录。

## 5. 年份与抉择

### 5.1 年份类型
- 普通年
- 异动年（小事件）
- 关键抉择年（milestone）

### 5.2 抉择触发规则
- `minEligibleAge` 前不触发（默认 `5` 岁前禁触发）
- 按年龄阶段概率随机：
  - 幼年：`0.1`
  - 青年：`0.2`
  - 壮年/中年/老年：`0.3`
- 连续 `guaranteeYears` 未触发保底触发（默认 `20` 年）
- 触发背景种子从 `data/events/faction-events.json` 对应世界事件池随机抽取

### 5.3 决策后的显示
- 时间线中展示该抉择年的叙事文本与属性变化
- “抉择历史”展示：
  - 年龄与阶段
  - 抉择背景
  - 选择项（A/B/C）
  - 掷点结果胶囊（来自该 milestone 年份 `statChanges`）

## 6. 结局与重开
1. 结局状态仅有：
   - `dead`（死亡）
   - `ascended`（飞升）
2. 结束后显示结算弹窗与结局文案
3. 点击“重开”后需重新确认 Setting

## 7. 常见问题
- `missing_game_environment_config`
  - 未先在 Setting 点击“确认本局环境”
- `deploy_mode_env_mismatch`
  - 启动链路与已确认会话模式不一致
- 本地模式开局失败
  - 检查本地 API Key 与模型配置
- 年份文本异常短或无输出
  - 检查模型可用性、超时、key
  - 空文本会触发后端 fallback 文案
