# 使用流程（v0.8.0）

## 1. 启动项目

### 1.1 本地链路（推荐开发）
1. 双击 `start-local.bat`
2. 脚本会自动：
   - 检查 Node/npm
   - 必要时执行 `npm install`
   - 自动补全 `apps/backend/.env`
   - 尝试释放 `5173/4000` 端口
3. 浏览器打开：`http://localhost:5173`

### 1.2 云端链路（体验）
1. 双击 `start-cloud.bat`
2. 确保后端环境变量里有 `CLOUD_MODEL_API_KEY`
3. 浏览器访问前端地址

## 2. 进入游戏前（Setting）
1. 点击左上角 `Setting`
2. 在“会话配置”页确认：
   - 当前部署链路（本地/云端）
   - 世界观、难度
   - 本地模式下填写 `API Key` 与模型配置
3. 点击“确认本局环境”
4. 状态变为“已确认”后关闭 Setting

## 3. 创建角色并开局
1. 输入人设提示词（至少 4 字）
2. 分配五维属性：
   - `智力/魅力/家境/气运/体魄`
   - 点数总和必须等于“本局可用天赋点”（20~30）
3. 翻牌并选择 1~3 张天赋卡
4. 点击“开始游戏”

## 4. 年份推进与抉择
1. 普通推进：点击“继续推进年份”
2. 当出现关键抉择时：
   - A：低风险低收益
   - B：中风险中收益
   - C：高风险高收益
3. 决策后系统继续推进，并实时展示：
   - 年龄阶段
   - 属性变化标签
   - 年度叙事文本
   - 名望与结局状态

## 5. 结局与重开
1. 结局状态仅有：
   - `死亡`
   - `飞升`
2. 结束后显示结局总结
3. 点击右上角“重开”可重置本局（需重新确认 Setting）

## 6. Setting 三个页签说明

### 6.1 会话配置
- 用途：仅影响“当前会话/当前局”
- 本地 key 不写盘

### 6.2 模型配置
- 用途：保存全局云端 Provider 参数到 `storage/runtime-config.json`
- 保存后对后续新开局生效

### 6.3 内容配置
- 用途：在线编辑 worlds/cards/difficulties/promptPack
- 保存后写入 `storage/custom-content.json`，并自动备份

## 7. 常见问题
- 提示“missing_game_environment_config”
  - 未先在 Setting 点击“确认本局环境”
- 提示“deploy_mode_env_mismatch”
  - 启动链路与会话环境不一致，重启并重新确认
- 本地模式提示缺少 key
  - 在会话配置里填写本地 API key
- 无叙事文本或很短
  - 检查模型可用性、超时、提示词约束与 key 是否有效
